# Отчет по выполнению задания

## Общие мысли

Были сложности выбора лучшей реализации, по крайней мере из тех, на что я способен.

Изначально планировалась реализация параметризированного интерфейса (парсер для типов данных CSV, JSON, XML). В итоге
принял решение не бежать за глубокой абстракцией и оставить это для будущих работ, там, где это действительно будет
нужно.

Создал отдельный класс — `EmployeeJsonXMLBuilder` с функционалом работы под разные типы данных.

## Проблемы

Изучал презентацию, пытался повторить код за автором (не переписывал «втупую», старался разбираться). Проблема
запоминания boilerplate-кода.

Например, взять механизм получения `Document` имеющегося XML-файла:

```java
DocumentBuilderFactory factory = DocumentBuilderFactory.newInstance();
DocumentBuilder builder = factory.newDocumentBuilder();
Document doc = builder.parse(new File("company.xml"));
```

Понятно, зачем тут фабрика, понятно, для чего `newDocumentBuilder`. А вот если капнуть глубже (с вопросом: *«А что это
за карусель объектов?»*), например рассмотреть метод `parse()`, вызываемый на `DocumentBuilder`, наталкиваешься на
архитектурный огород. Ничего не остается, кроме как запоминать процедуру получения экземпляра `Document`... а нейронка
говорит, что этот код вообще на уровне 2000-х годов, предлагая сторонние библиотеки.

**Основная боль:** то, что я тут писал два вечера, нейронка за минуту напишет, и я не понимаю, где моё место в мире IT.
